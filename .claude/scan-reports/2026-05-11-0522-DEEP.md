## CRITICAL findings (5) — Five Cs

### C1 — Command injection in `find_files` (sh -c with raw format!)
- **File**: `crates/cortex-tools/src/tools/search.rs:114-121`
- **Condition**: `Command::new("sh").arg("-c").arg(format!("find {search_path} -name '{pattern}' -type f 2>/dev/null | head -100"))`. Both `search_path` and `pattern` are LLM-controlled, raw-interpolated, unquoted, no `--`.
- **Cause**: format-string interpolation into a shell.
- **Criteria**: any prompt that gets WRITER to call `find_files` with `pattern="x'; rm -rf ~; #"` executes arbitrary code as daemon UID.
- **Consequence**: RCE as daemon user. Game over — daemon has filesystem and network.
- **Corrective**: drop the shell. `Command::new("find").arg("--").arg(search_path).args(["-name", &pattern, "-type", "f"])`.
- **Confirmed**: PROBE ✓ · **Effort**: XS (15 min) · **Agents**: SENTINEL

### C2 — Path traversal — apply.rs `validate_relative_path` + MCP twin (FN-1)
- **Files**: `crates/cortex-daemon/src/apply.rs:63-74` AND `crates/cortex-mcp/src/verification.rs:208`
- **Condition**: `validate_relative_path` checks `is_relative()` + no `ParentDir` only. No `canonicalize()`, no `symlink_metadata()`, no NUL check, no post-resolve `starts_with(workspace_root)`. MCP's `handle_apply_if_clean` writes to caller-supplied absolute path with NO workspace boundary check whatsoever.
- **Cause**: trust boundary not defined as a type. The "sandbox is a copy" defense does not apply to the real-disk write paths.
- **Criteria**: symlink in workspace pointing to `/etc/cron.d/cortex` or `~/.ssh/authorized_keys`. MCP client with arbitrary `file_path` payload.
- **Consequence**: privilege escalation via cron, ssh authorized_keys, shell rc, systemd user units. MCP variant is trivially weaponizable through any MCP client config.
- **Corrective**: introduce `WorkspaceGuard` / `WorkspacePath` newtype in cortex-core. Every public write API in `apply.rs`, `verification.rs`, `file_ops.rs` accepts only this type. Construction validates canonicalize + parent-starts_with(workspace) + reject if any ancestor is a symlink + reject NUL bytes.
- **Confirmed**: PROBE ✓ · **Effort**: S (2-3 hr) · **Agents**: SENTINEL + PHANTOM

### C3 — Cloud exfiltration default (S009 + FN-3 combined)
- **File**: `crates/cortex-core/src/config.rs:219-267`
- **Condition**: `cloud_enabled=true` by default. `CORTEX_OLLAMA_URL` env var overrides config with NO scheme/host validation. Cloud model names like `qwen3-coder:480b-cloud` route through `ollama_url`. Every prompt + full workspace context flows over plain HTTP to whatever `ollama_url` resolves to.
- **Cause**: defaults optimized for "works out of box" not "doesn't exfil out of box." Env override has no allowlist.
- **Criteria**: any env hijack (shell rc, MCP config, `.envrc` from a workspace), or an attacker who can write to `~/.config/cortex/`, redirects all prompts + file context to attacker endpoint. No log evidence at default verbosity.
- **Consequence**: silent workspace exfiltration. Source code, prompts, API keys, secrets — all leak.
- **Corrective**: `cloud_enabled=false` default. `[ollama] allowed_hosts = ["127.0.0.1", "localhost", "::1"]`. Refuse non-localhost without explicit `--allow-remote-ollama` flag. INFO-level log on every outbound request URL host.
- **Confirmed**: PHANTOM math ✓ · **Effort**: S (1-2 hr) · **Agents**: SENTINEL + PHANTOM

### C4 — `cargo check` 30s timeout fails OPEN (gate bypass)
- **File**: `crates/cortex-core/src/gate.rs:207-214, 322-365`
- **Condition**: `run_with_timeout(Duration::from_secs(30))` returns `None` on timeout. SandboxGate maps `None → Skipped → accepted=true` with reason "sandbox setup failed — skipping gate."
- **Cause**: timeout treated as "couldn't check" rather than "didn't verify."
- **Criteria**: any cold-cache cargo check, any slow proc-macro, any adversarial prompt that produces slow-compile code. Routine on first run.
- **Consequence**: **the daemon is theatrically verifying.** ADR-003's entire thesis ("verification-first gate") is silently violated on the wire. This finding alone invalidates the project's identity. An adversarial WRITER learns the timeout exists and generates slow code to bypass.
- **Corrective**: distinguish `Timeout` from `SpawnFailed`. Timeout → `VerificationResult { accepted: false, reason: "verification timed out" }`. Add `cargo check --offline --frozen --locked`. Share `CARGO_TARGET_DIR=/tmp/cortex-shared-target` to eliminate cold-cache. Add `verification_timeout_total` metric.
- **Confirmed**: PROBE ✓ · **Effort**: S (1 hr code; M with target/ cache plumbing, 4-6 hr) · **Agents**: MEDIC + PHANTOM upgrade

### C5 — `server.rs` god-file (2828 → 4315 LOC, 36 functions)
- **File**: `crates/cortex-daemon/src/server.rs`
- **Condition**: socket accept loop + all 5 request handlers (Ask, Apply, Debate, Implement, Research) + full adversarial debate engine + multi-phase research pipeline + plan parser/executor + symbol context builder + **DUPLICATE older tri-critic at lines 1315-1420** — all in one file. +1487 uncommitted lines pending.
- **Cause**: organic growth, no module boundary discipline, no review gate. server.rs is a private monolith with no escape hatch (`lib.rs` exposes only apply, critic, ollama).
- **Criteria**: every architectural drift in the project ends up in this file. ADR-004's "no critic" was lost here. ADR-003's language-agnosticism was lost here.
- **Consequence**: this file is the bottleneck for every future change. Merge conflicts will compound. Cognitive load is already blocking refactor velocity. **This is where ADRs go to die.**
- **Corrective**: extract into `server/{dispatch,ask,apply_handler,debate,implement,research}.rs`. Target server.rs ≤ 400 LOC = RPC entrypoint only. Delete the duplicate critic at lines 1315-1420 and have debate handlers import from `critic.rs`.
- **Confirmed**: PROBE ✓ (wc -l = 2828) · **Effort**: L (1-2 days) · **Agents**: SURVEYOR

## 5 Systemic Recommendations

### SR1 — Workspace Trust Boundary (one type, one source)
Define `WorkspaceGuard` in cortex-core. Every path-accepting public function across cortex-core, cortex-mcp, cortex-tools, cortex-daemon accepts only `&WorkspaceGuard` / `WorkspacePath` — never raw `&Path`. Compile-time enforcement. Single source of validation truth.

### SR2 — Verification-First Means Fail-Closed
Three outcomes only: `Pass`, `Fail`, `Indeterminate`. `Indeterminate` (timeout, missing toolchain, sandbox setup error) always maps to `accepted=false`. Add property test: random valid+invalid Rust files, assert `Indeterminate → !accepted`. This restores ADR-003 honesty.

### SR3 — Dev Infrastructure Floor (the 4-6 hour fix that pays back permanently)
- `rust-toolchain.toml` pinning channel 1.93
- `clippy.toml` with `disallowed-methods = ["Result::unwrap", "Option::unwrap"]` (lib crates)
- `deny.toml` for cargo-deny (licenses + advisories)
- `.github/workflows/ci.yml`: fmt + clippy + check + test + cargo-deny + cargo-audit on every push
- `rustfmt.toml` for consistency
- `CONTRIBUTING.md` — 1-page review checklist
- `README.md` — crate map + build instructions

### SR4 — Kill Critic OR Document It
Path A (preferred): delete `critic.rs` + server.rs duplicate. Honor ADR-004.
Path B: write ADR-005 superseding ADR-004 with the empirical reason critic was reintroduced; pick default-on OR opt-in consistently; delete the server.rs divergent fork either way.
Status quo: forbidden.

### SR5 — server.rs Decomposition Hardened
Cap any file in `crates/cortex-daemon/src/` at 600 LOC via clippy lint or custom rust-analyzer check. Without a mechanical cap, organic growth resumes after extraction.
